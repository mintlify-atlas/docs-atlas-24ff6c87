Tags are the primary filter mechanism on [analysis-tools.dev](https://analysis-tools.dev) and in the JSON API. Every tool must have at least one tag from `data/tags.yml`, or the renderer will reject it.

## Tag types

There are two kinds of tags:

| Type | Description |
|------|-------------|
| `language` | A programming language the tool analyzes (e.g. Python, Go, Rust). |
| `other` | A non-language category, framework, or artifact type (e.g. security, Kubernetes, Dockerfile). |

In `data/api/tools.json` these appear as two separate arrays on each tool entry:

- `languages` — contains the `value` strings of all `language` tags for that tool.
- `other` — contains the `value` strings of all `other` tags for that tool.

## Tag JSON structure

Each tag in `data/api/tags.json` has the following shape:

```json
{
  "name": "Python",
  "value": "python",
  "tag_type": "Language"
}
```

<ResponseField name="name" type="string" required>
  Human-readable display name shown on the website and in the README (e.g. `"Python"`, `"Security/SAST"`).
</ResponseField>

<ResponseField name="value" type="string" required>
  Machine-readable slug used in tool YAML files and in the JSON API arrays (e.g. `"python"`, `"security"`). This is the value you filter on in `jq` queries.
</ResponseField>

<ResponseField name="tag_type" type="string" required>
  Either `"Language"` or `"Other"`.
</ResponseField>

## Querying by tag

### Find all tools with a specific language tag

```bash
jq '[to_entries[]
  | select(.value.languages | index("rust"))
] | map(.key)' data/api/tools.json
```

### Find all tools with a specific "other" tag

```bash
jq '[to_entries[]
  | select(.value.other | index("security"))
] | map(.key)' data/api/tools.json
```

### Find tools matching both a language and an "other" tag

```bash
jq '[to_entries[]
  | select(.value.languages | index("python"))
  | select(.value.other | index("security"))
] | map(.key)' data/api/tools.json
```

### List all tags in use across the catalog

```bash
# All language values actually referenced by at least one tool
jq '[.[].languages[]] | unique | sort' data/api/tools.json

# All "other" values actually referenced by at least one tool
jq '[.[].other[]] | unique | sort' data/api/tools.json
```

<Tip>
  The full canonical tag list — including tags not yet used by any tool — lives in `data/api/tags.json`. The queries above only show tags that appear in at least one tool entry.
</Tip>

## Adding a new tag

New tags must be added to `data/tags.yml` before you can use them in a tool's `.yml` file. See the [contributing tags guide](/contributing/tags) for the process.
